# テストカバレッジパターン

## コンポーネントの場合

1. **レンダリング確認**: 正常にマウントされること
2. **Props による表示切り替え**: 各 Props パターンでの表示
3. **ユーザーインタラクション**: click, input, submit イベント
4. **状態表示**: ローディング / エラー / 空状態
5. **アクセシビリティ**: `getByRole` でアクセス可能であること

## API Route の場合

1. **各 HTTP メソッドの正常レスポンス**: ステータスコードとレスポンス形式
2. **バリデーションエラー（400）**: 不正な入力パターン
3. **認証エラー（401）/ 権限エラー（403）**: 未認証・権限不足
4. **存在しないリソース（404）**: 無効な ID
5. **サーバーエラー（500）**: DB 接続失敗等

## エッジケース（共通）

- `null`, `undefined` の入力
- 空配列 `[]`、空文字列 `""`
- 境界値（0、負数、最大値）
- 非常に長い文字列
