# コードレビュー

## コンテキスト（自動取得）

- ブランチ: !`git branch --show-current`
- 変更差分: !`git diff main --stat`
- 変更ファイル: !`git diff main --name-only`

## レビュー観点（重要度順）

1. **バグ・ロジックエラー**: 入力に関係なく誤った結果を返すコード
2. **セキュリティ**: SQLi, XSS, 認証バイパス, シークレット露出
3. **型安全性**: TS エラー, `as` キャスト, `any` 型の使用
4. **エラーハンドリング**: try/catch の欠落、エラー境界の不足
5. **パフォーマンス**: 不要な再レンダリング、N+1 クエリ、バンドルサイズ
6. **アクセシビリティ**: セマンティック HTML、ARIA、キーボード操作
7. **テストカバレッジ**: 新機能に対応するテストの有無

## 出力形式

重要度 80 以上の問題のみ報告する。各問題について:

```
📍 path/to/file.ts:42
🔴/🟡/🟢 重要度（Critical/Warning/Suggestion）
説明: 問題の具体的説明
修正案:
（コード例）
```

## 報告対象外

- スタイルの好み
- 一般的なコード品質の感想
- 「こうした方が良いかも」レベルの提案
