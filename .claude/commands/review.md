## コンテキスト

- 現在のブランチ: !`git branch --show-current`
- 変更差分: !`git diff main --stat`
- 変更ファイル一覧: !`git diff main --name-only`

## レビュー観点

以下の観点で変更差分をレビューし、問題を**重要度順**に報告してください：

1. **バグ・ロジックエラー**: 入力に関係なく誤った結果を返すコード
2. **セキュリティ**: SQLインジェクション、XSS、認証バイパス、機密情報の露出
3. **型安全性**: TypeScript の型エラー、as キャスト、any 型の使用
4. **エラーハンドリング**: try/catch の欠落、エラー境界の不足
5. **パフォーマンス**: 不要な再レンダリング、N+1クエリ、バンドルサイズ
6. **アクセシビリティ**: セマンティックHTML、ARIA属性、キーボード操作
7. **テストカバレッジ**: 新機能に対応するテストの有無

## 出力形式

各問題について：

- 📍 ファイルパスと行番号
- 🔴/🟡/🟢 重要度（Critical/Warning/Suggestion）
- 問題の具体的説明
- 修正案のコード例

重要度80以上の問題のみ報告し、スタイルの好みや一般的なコード品質の指摘は除外してください。
